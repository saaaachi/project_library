# 2. プロジェクト概要

## プロジェクト名

Project Library

（開発コードネーム）

---

## サイトコンセプト

無料でダウンロードできる
脳トレ・知育プリントを公開し、

利用者が探しやすく、

運営者が続けやすい

「作品管理システム付き無料プリント図書館」

を作る。

単なるダウンロードサイトではなく、

何度でも戻ってきたくなる
"図書館"を目指す。

---

## Project Libraryが目指すもの

Project Libraryは

作品を公開することが目的ではない。

利用者が

「また来よう」

と思える場所を作ることが目的である。

また、

運営者にとっても

「投稿が面倒だから更新しない」

という状態を作らないことを重視する。

サイトは

利用者にも

未来の運営者にも

優しい設計を目指す。

---

# 3. 運営目標

Project Libraryでは
次の目標を基本方針とする。

・スマホだけでも投稿できる

・投稿時間3分以内

・作品数が増えるほど資産になる

・長期間運営できる

・広告収益で運営費以上を目指す

・利用者が探しやすいサイトにする

・1000作品以上になっても管理しやすい

・改善を続けながら育てる

---

# 4. 開発理念

Project Libraryでは

「未来の自分が困らない」

ことを最優先とする。

コードを書くことよりも

続けられる仕組みを作ることを大切にする。

また、

利用者だけではなく

運営者も使いやすいサイトを目指す。

---

# 5. 基本ルール

Project Libraryでは

以下を開発ルールとする。

・固定費はできるだけ0円

・設計書を先に更新する

・設計書完成後にコードを書く

・コードは共通化を優先する

・同じコードを書かない

・スマホだけでも管理できる

・修正は一か所だけで済む設計を目指す

・作品が1000件以上になっても困らない設計を維持する

・後から機能追加しやすい構造を作る

---

# 6. Project Library 開発方針

Project Libraryは

完成させることではなく

育て続けることを目的とする。

Version1.0で開館し、

Version1.1

Version1.2

Version2.0

そのため、

大きく作り直すことは避け、

改善を積み重ねる。

# 36. Project Library憲章

Project Libraryは

作品を増やすことだけを目的としない。

利用者が

安心して利用でき

何度でも訪れたくなる図書館を目指す。

また

未来の運営者である自分自身も

楽しく続けられる仕組みを作る。

新しい機能を追加するときは

必ず設計書を更新し

設計書に沿って実装する。

積み重ねた作品は

未来の資産となる。

Project Libraryは

利用者にも

未来の自分にも

優しい図書館であり続ける。
